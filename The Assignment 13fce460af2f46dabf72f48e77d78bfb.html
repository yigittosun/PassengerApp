The Assignment

Front-end Developers

Introduction

Congratulations for making it this far!

We believe that the best way to gauge a person’s skills is to inspect their work on a specific project so we have formulated an assignment to test if your technical knowledge meets our business expectations. The assignment gives the person a chance to show their understanding of the area they are/want to specialize in, regardless of their social or personal skills that could interfere during an in-person interview. You can directly reach us for your questions. Our Software Team Lead is Egemen Yildiz (egemen@voltlines.com), let him know if you have something is not understood well.

Case

In this case study, user expects to create, view and interact with a routing solution from point A (origin) to point B (destination) including several passengers and their pick-ups. Given an array of passengerInfo objects, we expect your finished project to cover expectations and maybe more.

Conditions

  • We expect you to use Google Maps Platform to do map related operations
  • Vue.js is heavily used in most of our products currently but you are free to use any framework of your choice
  • You have 7 days to finalize your assignment
  • Please do not use any company related assets, names or wordings in your project

Input

The user should have the ability to input an array of passengerInfo objects. This can be a long text-area which the user can paste test passengerInfo objects as text, or any kind of input you would provide for the user as long as it is structured in the same way. Including a request/response cycle (even if it's mock data) to fetch user input is the preferred approach.

Technical & Visual Expectations

Below are our expectations and We expect you to visualize at least 3 components that are showing:

  1. Map View
  1. Table View
  1. Route Detail View

Please complete as much items as you can within the given time.

Passenger Enrichment & Information

We expect you to enrich input data with the following fields:

  • pickUpPointOrder (integer) order of passenger's pick-up on route
  • tripDuration (number) passenger's trip duration from her pickUpPoint to destination in minutes

With enriched data, the user should be able to,

  • see (static, not draggable) markers for passenger's pickUpPoint
  • view, sort & filter items of a table which shows passengerInfo.name, tripDuration, pickUpPointOrder

Route Information

The user should be able to,

  • put a marker on the map as origin
  • put a marker on the map as destination
  • see polyline between origin & destination of route
  • total route distance (from origin to destination) in kilometers
  • total route duration (from origin to destination) in hours
  • average of passengers' tripDuration

Validations

The user should be informed with a warning message pop-up when following validations didn't pass:

  • Route duration from start to finish is over 2 hours
  • Input has passengers more than 9
